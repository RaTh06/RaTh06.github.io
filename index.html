图床教程

发表于 2026-01-16 分类于 教程
本文字数： 1.7k 阅读时长 ≈ 2 分钟

推荐方案：Cloudflare R2 + PicGo

这是目前最推荐给使用了 Cloudflare 用户的方案。

  • 为什么选它？
    1. 完全免费：Cloudflare R2 提供 10GB 的免费存储空间（对博客图片来说几乎用不完）和 零流量费。
    2. 速度快：直接利用 Cloudflare 全球 CDN 加速。
    3. 无需备案：你的域名在 spaceship 买的，DNS 在 Cloudflare，完全不需要经过国内备案流程。
    4. 专业：你可以使用自己的二级域名（如 img.你的域名.com）作为图片链接。如果未来你想换图床，只需要改一下 DNS 解析，文章里的几百张图片链接完全不用动。

配置保姆级教程

第一步：在 Cloudflare 开启 R2 存储桶

  1. 登录 Cloudflare Dashboard。
  2. 在左侧菜单找到 R2（如果没开通，点击启用，需要绑定信用卡或 PayPal 验证，但不会扣费，只要你没超过 10GB）。
  3. 点击 “Create bucket” (创建存储桶)。
    • Bucket name: 随便起，比如 blog-imgs。
    • Location: 建议选 Automatic 或 North America (因为你的博客在 GitHub Pages，美西节点连接顺畅)。
    • 点击 Create bucket。

第二步：绑定你的域名 (关键)

  1. 进入刚才创建的 bucket 页面。
  2. 点击顶部的 “Settings” (设置) 标签。
  3. 向下滚动找到 “Public Access” (公开访问) -> “Custom Domains” (自定义域)。
  4. 点击 “Connect Domain”。
  5. 输入你想用的二级域名，比如 img.你的域名.com (前提是你的主域名 DNS 确实是在 Cloudflare 管理的)。
  6. 点击 Continue -> Connect Domain。
  7. 等待几分钟，状态显示 “Active” 后，以后你的图片就可以通过 https://img.你的域名.com/图片名.png 访问了。

第三步：获取 API 密钥

为了让 PicGo 能上传图片，你需要“钥匙”。

  1. 回到 R2 的主界面（列出所有 bucket 的那个页面）。
  2. 在右侧找到 “Manage R2 API Tokens”。
  3. 点击 “Create API Token”。
  4. Permissions: 选择 Admin Read & Write (读写权限)。
  5. TTL: 选 Forever (永久)。
  6. 点击 Create API Token。
  7. 重要： 屏幕上会显示 Access Key ID 和 Secret Access Key，立刻复制保存下来，这个页面关了就再也看不到了。

第四步：配置 PicGo

PicGo 原生不支持 R2，需要安装一个插件（因为 R2 兼容 AWS S3 协议）。

  1. 打开 PicGo -> 插件设置。
  2. 搜索安装 picgo-plugin-s3 (注意是 s3，不是 aws-s3，推荐用 s3 插件更轻量，或者 picgo-plugin-aws-s3 也可以)。
    • 如果安装失败，可能需要先安装 Node.js，或者多试几次。
  3. 安装好后，点击 图床设置 -> Amazon S3 (插件装好后会出现这个选项)。
  4. 填入配置：
    • 图床配置名: R2 (随便填)
    • 应用密钥 ID (AccessKeyID): 刚才复制的 Access Key ID。
    • 应用密钥 (SecretAccessKey): 刚才复制的 Secret Access Key。
    • 桶名 (Bucket): 你创建的 bucket 名字 (比如 blog-imgs)。
    • 文件路径: {year}/{month}/{fileName} (可选，按年月分类)。
    • 地区 (Region): auto (如果你一定要填，填 us-east-1，但在 Cloudflare R2 中通常填 auto 或留空)。
    • 自定义节点 (Endpoint): 这是关键！
      • 回到 Cloudflare R2 界面，在 bucket 详情页，你会看到一个 “S3 API” 的链接，通常长这样：https://<你的账号ID>.r2.cloudflarestorage.com。
      • 把这个链接填进去。 (注意：不要在后面加 bucket 名字)。
    • 自定义域名: https://img.你的域名.com (刚才绑定的那个二级域名)。
    • 强制路径样式 (ForcePathStyle): 选 否 (或者如果不成功改选 是 试试)。
  5. 点击 确定 并 设为默认图床。
