数据结构

1 绪论

1.1 数据结构的基本概念

1.数据：数据就是信息的载体，是对客观事物属性的描述。

2.数据元素：数据的最小单位，通常由值域(存储的数据)和相关属性(标识符或下标)组成。

3.数据对象：有相同性质的数据元素的集合，形成一个整体。

4.数据类型：一个值的集合及其允许的操作，分为原子类型、结构类型和抽象数据类型(ADT)。

5.数据结构：相互之间存在一种或多种特定关系的数据元素的集合。包括逻辑结构、存储结构、和数据的运算。

1.2 数据结构三要素

逻辑结构：定义了数据元素之间的逻辑关系，不考虑计算机真实的内存布局。它独立于存储结构，可分为线性结构和非线性结构。

四种基本结构关系如下图：

存储结构：在计算机内存中的具体映射。同样的算法设计用不同的存储实现效率完全不同。如栈可以用顺序存储(数组实现)，也可以使用链式存储(链表实现)。

常见的存储结构有顺序存储、链式存储、索引存储和散列存储。

顺序存储：逻辑上相邻的数据元素存储在物理上相邻的存储单元中。优点是随机存储，时间复杂度为O(1)，缺点是容易产生外部碎片(要使用连续的存储单元)。

链式存储：不要求逻辑相邻的数据元素存储在物理上相邻的存储单元。使用指针指示元素的存储地址。优点是不会产生碎片，充分利用存储单元。缺点是每个元素存储指针而额外占用空间，而且只能顺序访问。

索引存储：在存储数据元素时，额外建立索引表。索引表的每一项称为索引项，包括关键字和地址。优点是检索速度快，缺点是需要额外存储空间用于索引表，在增加和删除元素时需要更新索引表，带来额外时间开销。

散列存储：根据元素的关键字直接计算出其存储地址，称HashMap。优点是检索、插入和删除操作都很快速，缺点是散列函数如果设计不当，可能产生冲突，解决冲突会增加时间和空间成本。

数据的运算：包括定义和实现。定义说明运算的功能，实现完成运算的具体步骤。

1.3 算法和算法评价

算法是对特定问题求解步骤的描述，是一个有穷指令序列。一个好的算法应该具有下面五个重要特性。

有穷性：算法在有限步后结束，每一步在有限时间内完成。

确定性：算法每条指令有明确的含义，对于相同的输入，只能产生相同的输出。

可行性：算法描述的操作可以通过已实现的基本运算执行有限次数完成。

输入：算法可以有零个或多个输入。

输出：算法至少有一个输出。

设计一个好的算法，应该满足以下目标：

  1. 正确性
  2. 可读性
  3. 健壮性
  4. 高效率和低存储量需求

算法效率的度量：在算法设计中，正确性是基础，效率是衡量算法优劣的关键指标。我们引入数学中渐近复杂度分析——通过时间复杂度和空间复杂度来描述算法随问题规模增长的资源消耗趋势。

时间复杂度：用大O记法，不关注算法所用的时间，只关注其所用的步数。它反映的是当数据量(问题规模)增大时，步数如何变化。通常只要关注大O中增长最快的项，且考虑算法的最坏情况。

空间复杂度：表示算法在运行过程中所需要的额外存储空间，若算法只用常量级的额外空间，空间复杂度为O(1)。

循环执行次数等于 i 从 1 以乘 2的方式增长，直到 i > n。

  • 第 2 次：i = 2 → 4
  • 第 3 次：i = 4 → 8
  • …
  • 第 k 次：i = 2^(k-1) → 2^k

循环结束条件：2^k > n

1.5 斐波那契数列

分析步骤：

  1. 建立递推关系：
    设 T(n) 为 fib(n) 的基本操作次数：
    T(n) = T(n-1) + T(n-2) + 1
    其中 +1 表示本层加法操作。
  2. 展开递归：
    T(n) = T(n-1) + T(n-2) + 1
    = (T(n-2) + T(n-3) + 1) + T(n-2) + 1
    = 2T(n-2) + T(n-3) + 2
    继续展开，可以看到执行次数呈指数增长。
  3. 时间复杂度：
    T(n) = O(2^n)
