hexo部署与域名解析

发表于2026-01-17|更新于2026-01-17|教程
|总字数:223|阅读时长:1分钟|浏览量:

创建GitHub仓库

  • 新建仓库：在 GitHub 上创建一个名为 rath06.github.io 的公开仓库。
  • 获取地址：点击 Code 按钮，复制 SSH 链接：git@github.com:rath06/rath06.github.io.git。

域名解析 (Cloudflare)

  • CNAME 记录 1：添加名称为 @ 的 CNAME 记录，目标指向 rath06.github.io。
  • CNAME 记录 2：添加名称为 www 的 CNAME 记录，目标指向 rath06.github.io。
  • SSL 模式：在 SSL/TLS 选项中，将模式设置为 “完全 (Full)”。

本地环境配置

  • CNAME 文件：在 D:\Blog\source\ 目录下新建文件 CNAME（无后缀），内容写入 rath02.top。

  • 修改站点配置：打开博客根目录 _config.yml，修改以下字段：

    • url: https://rath02.top

    • deploy 部分修改为：

  • 安装插件：在终端运行 npm install hexo-deployer-git --save。

部署命令

在博客根目录依次执行：

  1. hexo clean
  2. hexo g
  3. hexo d
